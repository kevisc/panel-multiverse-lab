Data Generating Process (DGP)

The simulated data follows this model:

yit = α + β·Dit + ui + εit + γ·t + measurement error where: • yit = outcome for individual i at time t • Dit = binary treatment indicator (0 or 1) • β = true causal effect (what we want to estimate) • ui = individual fixed effect (time-invariant, unobserved) • εit = idiosyncratic error (varies by person and time) • γ·t = time trend (if enabled) Selection mechanism: P(Dit = 1) = logistic(α + ρ·ui/σu) When ρ > 0, individuals with higher ui are more likely to receive treatment, creating omitted variable bias in simple OLS.
The Four Estimators Compared
EstimatorWhat It DoesKey AssumptionWhen It Works
Cross-Section OLS Regress Y on D using wave 1 only E[ui|Di] = 0 No selection on unobservables
Pooled OLS Stack all waves, ignore panel structure E[ui|Dit] = 0 No selection; SEs may be wrong
Fixed Effects Within-transformation: demean by individual E[εit|Dit, ui] = 0 Selection on time-invariant factors OK
Random Effects GLS with quasi-demeaning (θ) E[ui|Dit] = 0 (RE exogeneity) No correlation between ui and D
Key insight: FE eliminates all time-invariant confounding (observed and unobserved) by using only within-person variation. The cost: we can't estimate effects of time-invariant variables, and we need within-person variation in treatment.
Understanding the Hausman Test

The Hausman test compares FE and RE estimates:

H = (βFE - βRE)² / (Var(βFE) - Var(βRE)) Under H₀: RE is consistent (no correlation between ui and Dit) Under H₁: RE is inconsistent, FE is preferred

Interpretation:

  • Large H (small p-value): Reject H₀ → use FE
  • Small H (large p-value): Can't reject → RE may be more efficient
  • FE is always consistent; RE is more efficient when valid
Sources of Bias
ProblemEffect on OLSSolution
Selection on unobservables (ρ > 0) Upward or downward bias FE, IV, or design-based ID
Time trends Spurious correlation if D trends Include time fixed effects
Measurement error in Y Larger SEs, no bias in β Better measures, larger N
Measurement error in D Attenuation bias toward 0 IV, validation studies
Reverse causality Bias direction depends on sign Lagged D, IV, timing
Practical Recommendations
  1. Always start with descriptives: Plot the data, understand variation patterns
  2. Compare estimators: Large differences suggest selection bias
  3. Run Hausman test: Formally test FE vs RE
  4. Check diagnostics: Residual plots reveal model misspecification
  5. Use cluster-robust SEs: Standard OLS SEs ignore within-person correlation
  6. Consider time effects: Add wave dummies if outcomes trend over time
  7. Report multiple specifications: Sensitivity analysis builds credibility

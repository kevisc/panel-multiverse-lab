# R (plm) replication of the Panel Data Multiverse Lab

Re-fit of every specification in the default multiverse (3 estimands x 480 specs = 1440) using R's `plm`, compared to the browser tool's JavaScript engine.

- **Identification agreement** (whether the focal coefficient is estimable): **1436 / 1440** (99.7%).
- **Coefficient agreement** on the 1244 specifications identified by both engines, by estimator:

| Estimator | n | max |Δβ| | mean |Δβ| |
|---|---|---|---|
| pooled | 288 | 9.25e-07 | 3.30e-07 |
| between | 288 | 1.01e-06 | 3.55e-07 |
| within | 192 | 2.27e-06 | 9.35e-07 |
| fd | 192 | 2.04e-06 | 1.09e-06 |
| random | 284 | 3.04e-04 | 4.20e-05 |

All five estimators reproduce R's `plm`: pooled OLS, between, fixed-effects and first-difference coefficients to ~1e-6 (numerical precision), and random effects to ~3e-4 (a Swamy-Arora variance-component estimator). The 4 identification exceptions are all random-effects specifications whose only regressor is time-invariant (e.g. schooling with no other covariates): `plm` declines to fit them because there are no within degrees of freedom to estimate the idiosyncratic variance, whereas the tool returns the feasible-GLS estimate. Generated by `verification/replicate_plm.R`.
